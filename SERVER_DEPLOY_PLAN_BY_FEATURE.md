# 서버 업데이트 실행 계획 (기능 기준)

작성일: 2026-03-05  
목표: 변경 기능을 안전하게 반영하고, 배포 직후 즉시 접속 가능한 상태를 보장한다.

---

## 1) 이번 배포 기능 요약 (파일 기준 아님)

1. 관리자용 로그 리뷰 기능 추가
   - 관리자 전용으로 세션 목록 조회
   - 세션별 대화 로그 조회
   - 북마크/코멘트 저장
2. 결제 정책/표시 금액 변경
   - 결제/입금 관련 기본 금액을 7,900원 기준으로 일치화
3. 결제 진입 UX 변경
   - 무통장입금, 추천인코드, 결제위젯 선택 플로우로 분기
4. 추천인코드 기반 임시 Pro 접근
   - 코드 입력 시 일정 기간 Pro 접근 허용
5. 앱/웹 노출 정리
   - 앱에서 하단 사업자 정보 노출 정책 조정
   - 약관/정책 링크 오픈 방식 정리

---

## 2) 배포 전 핵심 리스크

### A. 결제위젯 경로 리스크 (최우선)
- 현재 운영에서 `/payments/*` 응답이 404일 수 있음.
- 원인 후보
  - `/payments`를 처리하는 별도 서버가 해당 경로를 미제공
  - 프록시 경로와 실제 정적 경로 설계 불일치
- 결론: 배포 전 반드시 `/payments/checkout.html`, `/payments/billing.html` 실응답(200) 확인 필요

### B. 관리자 리뷰 기능 의존성
- 관리자 리뷰 기능은 관리자 권한/저장소 접근이 필요함.
- 권한 또는 저장소 스키마가 맞지 않으면 관리자 화면 일부 동작 실패 가능

### C. 서비스 재시작 누락 리스크
- 백엔드 기능 추가가 포함되어 있어 백엔드 재기동 누락 시 구버전 동작 가능
- 프론트 빌드 누락 시 최신 UI/동선 반영 안 될 수 있음

## 3) 실행 계획 (운영 적용 순서)

## 단계 0. 배포 대상 고정
- 배포 대상 커밋/브랜치를 확정한다.
- 로컬에만 있는 수정(예: 결제 fallback 경로 조정)이 있으면 먼저 커밋/푸시하여 배포 대상에 포함시킨다.

완료 기준:
- 원격 브랜치 기준 단일 배포 대상이 확정됨

## 단계 1. 서버 현황 리스트업 (변경 전 스냅샷)
- 현재 브랜치/헤드 커밋/원격 대비 앞뒤 상태
- 워킹트리 상태(수정/미추적)
- 활성 프로세스(웹, API, 결제 관련)
- 최근 로그 에러 유무

완료 기준:
- "지금 서버가 어떤 상태인지"를 문장 4줄 내로 요약 가능

## 단계 2. 안전 업데이트
- fast-forward 방식으로 코드 동기화
- 자동 merge 없이 진행(충돌 시 즉시 중단)

완료 기준:
- 서버 HEAD = 원격 배포 대상 커밋
- 예상치 못한 로컬 수정 없음

## 단계 3. 프론트 반영
- 프론트 의존성 상태 확인 후 빌드
- 산출물 생성 확인

완료 기준:
- 최신 프론트가 서비스 경로에 반영됨
- 메인 진입 페이지 및 관리자 진입 화면 정상 노출

## 단계 4. 백엔드 반영
- 백엔드 프로세스 재시작
- 기동 확인(포트/헬스/로그)

완료 기준:
- API 응답 정상
- 관리자 리뷰/결제 관련 API 라우트 접근 가능

## 단계 5. 결제위젯 경로 검증 (필수 게이트)
- 아래 URL 직접 확인
  - `/payments/checkout.html`
  - `/payments/billing.html`
- 200 응답 아니면 배포 완료 처리 금지

완료 기준:
- 두 경로 모두 실접속 가능

## 단계 6. 통합 스모크 테스트
- 일반 사용자 채팅 진입
- 관리자 리뷰 화면 진입/목록 로드
- 결제 모달 진입 및 버튼 동선 확인
- 오류 로그 급증 여부 5분 관찰

완료 기준:
- 치명 오류 없음
- 즉시 사용자 접속 가능한 상태

---

## 4) 롤백 계획

롤백 트리거:
- 결제위젯 경로 404 지속
- 관리자 기능 호출 시 5xx 지속
- 기본 채팅/로그인 동선 장애

롤백 절차:
1. 직전 정상 커밋으로 되돌림
2. 프론트 재빌드
3. 백엔드 재시작
4. 핵심 URL/API 재검증

완료 기준:
- 배포 전과 동일한 정상 서비스 상태 복구

---

## 5) 오늘 배포 시 운영 판단 기준

배포 진행 가능:
- 결제위젯 경로 200 확인
- 프론트/백엔드 반영 후 핵심 동선 정상
- 에러 로그 급증 없음

배포 보류:
- 결제위젯 경로 404
- 관리자 리뷰 API 권한/스키마 오류 지속
- 백엔드 재기동 후 API 불안정
