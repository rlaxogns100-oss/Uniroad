## 1. 지금 뭘 볼 수 있나요?

PostHog에 **"Uniroad Golden Path"** 라는 대시보드가 하나 있고,  
그 안에 그래프(인사이트) **11개**가 들어 있습니다.

### 유입 (사람들이 어디서 오는가?)

| 그래프 이름 | 형태 | 뭘 보여주나요? |
|---|---|---|
| **Daily Visitors** | 선 그래프 | 날짜별 방문자 수 추이 |
| **Traffic Source Mix** | 원형 그래프 (파이 차트) | 어디서 들어왔는지 비율 (인스타, 블로그, 광고 등) |
| **Device Mix** | 원형 그래프 (파이 차트) | 모바일 vs PC 비율 |
| **First Action Path** | 경로 시각화 | 사이트에 들어와서 첫 번째로 하는 행동이 뭔지 |
| **Active Traffic Ratio** | 퍼널 | 방문한 사람 중 실제로 채팅까지 해본 사람의 비율 |

### 활성화 (들어온 사람이 뭘 하는가?)

| 그래프 이름 | 형태 | 뭘 보여주나요? |
|---|---|---|
| **Activation Funnel** | 퍼널 | 방문 → 채팅 입력 → 로그인까지 이어지는 비율 |
| **Signup Funnel** | 퍼널 | 방문 → 회원가입 창 봄 → 실제 가입 완료까지의 비율 |
| **Reverse Path to Signup** | 경로 시각화 | 회원가입한 사람이 **직전에 뭘 했는지** 거꾸로 추적 |

### 결제 (돈을 내는가?)

| 그래프 이름 | 형태 | 뭘 보여주나요? |
|---|---|---|
| **Revenue Funnel** | 퍼널 | 결제 안내 봄 → 결제 버튼 클릭 → 실제 결제 완료까지의 비율 |

### 유지 (돈 낸 사람이 계속 쓰는가?)

| 그래프 이름 | 형태 | 뭘 보여주나요? |
|---|---|---|
| **Feature Usage (Paid Users)** | 선 그래프 | 돈 낸 사용자가 어떤 기능을 많이 쓰는지 |
| **Paid User Retention** | 리텐션 표 | 돈 낸 사용자가 다음 주에도 다시 오는지 |

## 2. 각 그래프로 어떤 질문에 답할 수 있나요?

### 사람들이 어디서 오는가? (유입)

- **Daily Visitors** → "오늘 방문자가 어제보다 늘었나?"
- **Traffic Source Mix** → "인스타 광고랑 블로그 중에 뭐가 효과 있나?" (원형 그래프로 비율 한눈에)
- **Device Mix** → "우리 사용자 대부분이 모바일인가 PC인가?" (원형 그래프로 비율 한눈에)
- **First Action Path** → "처음 들어온 사람이 뭘 먼저 하나? 바로 채팅 치나, 스크롤만 하나?"
- **Active Traffic Ratio** → "들어와서 아무것도 안 하고 나간 사람이 얼마나 되나?"

### 들어온 사람이 뭘 하는가? (활성화)

- **Activation Funnel** → "방문만 하고 나가는 사람이 얼마나 되나?"
- **Activation Funnel** → "채팅까지 쳐본 사람 중에 로그인까지 하는 비율은?"
- **Signup Funnel** → "회원가입 창을 본 사람 중 실제로 가입한 비율은?"
- **Reverse Path to Signup** → "가입한 사람들은 가입 직전에 뭘 했나? 채팅? 생기부? 점수 입력?"

### 돈을 내는가? (결제)

- **Revenue Funnel** → "결제 안내를 본 사람 중 실제 결제한 비율은?"
- **Revenue Funnel** → "'질문 제한'이랑 '생기부 분석' 중 뭐 때문에 더 많이 결제하나?"

### 돈 낸 사람이 계속 쓰는가? (리텐션)

- **Feature Usage (Paid Users)** → "결제한 유저는 주로 생기부를 쓰나? 점수 연동을 쓰나?"
- **Paid User Retention** → "결제한 유저가 1주 뒤에도 다시 오나? 일회성인가?"

## 3. PostHog가 기록하는 사용자 행동 목록

사용자가 사이트에서 뭔가를 할 때마다, 아래처럼 기록이 남습니다.  
이 기록들이 쌓여서 위의 그래프가 만들어집니다.

### 페이지 방문

| 기록 이름 | 언제 남나요? |

| 페이지 방문 | 사용자가 어떤 페이지든 열 때 |
| 로그인/회원가입 창 노출 | 로그인 또는 회원가입 팝업이 뜰 때 |
| 결제 안내 노출 | "프로로 업그레이드하세요" 같은 결제 안내가 뜰 때 |

| 카카오/구글 로그인 클릭 | 소셜 로그인 버튼을 누를 때 |
| 로그인 완료 | 실제로 로그인에 성공할 때 |

## 6. 지금 주의할 점

1. **데이터가 아직 적습니다**  
   대시보드는 만들어졌지만, 의미 있는 패턴이 보이려면 최소 3~7일은 데이터가 쌓여야 합니다.

2. **운영 서버에 추가 설정이 필요합니다**  
   지금은 개발 환경에서만 PostHog 데이터 전송이 확실하게 작동합니다.  
   실제 운영 서버(uni2road.com)에서 안정적으로 데이터를 보내려면, 서버 설정(nginx)에 프록시를 추가해야 합니다.

3. **내부 계정(팀원)을 제외하고 봐야 합니다**  
   분석할 때 "우리 팀원이 접속한 데이터"가 섞이면 숫자가 왜곡됩니다.  
   PostHog에서 필터에 `is_internal` 이 아닌 것만 보도록 설정하면 됩니다.

4. **그래프 위치는 직접 정리할 수 있습니다**  
   대시보드 안에서 그래프 타일을 드래그해서 원하는 순서로 배치할 수 있습니다.
